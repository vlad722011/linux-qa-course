# linux-qa-course

Личный учебный проект: Linux для QA-инженера до уровня уверенного middle+ через практику на реальном VPS.

Идея простая: не “знать команды”, а уметь расследовать проблемы (логи, процессы, сеть, права, сервисы), фиксить и объяснять почему так.

Структура репозитория:
- cheatsheet/ — короткая шпаргалка: команды + когда применять + частые ключи.
  Содержание cheatsheet:
  - [02 — Pipes + redirects](cheatsheet/02-pipes-redirects.md)
- guide/ — учебник для чтения: объяснения, примеры, типовые ошибки, лабораторные, контрольные вопросы.

Правило обучения:
Объяснение → практика руками → проверка понимания → мини-инцидент (сломал/починил) → фиксация выводов.

Текущий фокус:
- Module 01: shell + streams (stdin/stdout/stderr)
- Далее: pipes + log investigation (auth.log / journalctl)
