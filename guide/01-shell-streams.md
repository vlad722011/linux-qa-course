Ключевая идея Linux:

Почему это важно QA:
• можно смотреть поведение приложения на уровне системы

Ключевая мысль главы:

2) Терминал, shell и сессия
Терминал — это интерфейс:
• окно в GUI
• SSH-клиент
• вкладка в IDE

Терминал сам ничего не делает — он просто передаёт ввод shell’у.

Shell — это программа (обычно bash), которая:
• читает команду
• создаёт процесс
• настраивает stdin/stdout/stderr
• управляет окружением
• возвращает код завершения (0 = успех)

Сессия — это отдельное подключение:
• свой текущий каталог
• свои переменные окружения
• свои процессы

Каждое SSH-подключение = новая сессия

## 3) Потоки данных
У процесса есть 3 стандартных потока:  
**stdin (0)** — вход (клавиатура или другой процесс)  
**stdout (1)** — обычный вывод  
**stderr (2)** — ошибки  

QA-почему важно:  
• В CI stdout и stderr могут идти отдельно  
• Можно сохранять ошибки отдельно для анализа

## 5) Пайпы (|)
Соединяем stdout одной команды с stdin другой.

Пример:
~~~bash
cat /var/log/syslog | grep ssh | tail -n 20
~~~

Что происходит:
• cat читает файл → stdout  
• grep фильтрует поток  
• tail берёт последние строки
