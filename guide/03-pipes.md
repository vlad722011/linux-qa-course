# 03 — Pipes (конвейеры) для QA

Цель: уметь быстро “расследовать” текстовый вывод команд и логи через конвейер: фильтр → трансформация → агрегация → топ.

## 1) Базовая модель: stdin/stdout/stderr
- `stdin` — входной поток.
- `stdout` — обычный вывод команды.
- `stderr` — ошибки (диагностика).
- Pipe `|` передаёт **stdout** слева в **stdin** справа.

Практический вывод: пайпы идеально подходят для логов/текста, но ошибки (`stderr`) по pipe не идут, пока ты их явно не перенаправишь (это будет в следующей теме про redirects).

## 2) Каркас “расследования”
Почти любая задача по логам/тексту укладывается в каркас:

1) Получить поток строк: `cat`, `journalctl`, `ps`, `ss`, `docker logs`…
2) Отфильтровать нужное: `grep`.
3) Извлечь поле: `awk`.
4) Сгруппировать и посчитать: `sort | uniq -c`.
5) Отсортировать топ: `sort -nr | head`.

Пример (шаблон):
cmd | grep "pattern" | awk '{print $NF}' | sort | uniq -c | sort -nr | head

## 3) Почему порядок важен (ключевая грабля)
`uniq` считает только **соседние** одинаковые строки. Поэтому для корректного подсчёта почти всегда нужен `sort` перед `uniq -c`.

Мини‑демо:
printf "a\nb\na\n" | uniq -c
# 1 a
# 1 b
# 1 a

printf "a\nb\na\n" | sort | uniq -c
# 2 a
# 1 b

## 4) Полезные паттерны (минимальный набор)

### Фильтрация
- Case-insensitive:
cmd | grep -i "error"
- Исключить строки:
cmd | grep -v "timeout"

### Взять конкретную строку
- 4-я строка:
cmd | head -n 4 | tail -n 1

### Последнее поле
cmd | awk '{print $NF}'

### Топ значений
cmd | sort | uniq -c | sort -nr | head -n 10

## 5) Мини‑контроль (5 задач)
Реши и объясни порядок:

1) `printf "ok\nerror: one\nok\nerror: two\nok\n"` → вывести только `error: two`.
2) `printf "a\nb\na\na\nb\nc\n"` → посчитать частоты и отсортировать по убыванию.
3) `printf "Feb 19 sshd Invalid user admin\nFeb 19 sshd Invalid user root\nFeb 19 sshd Invalid user admin\n"` → вывести топ логинов.
4) `printf "INFO started\nWARN disk\nERROR failed\nerror timeout\n"` → вывести error без регистра, исключить timeout.
5) На своём сервере: топ `Invalid user` за сегодня через `journalctl -u ssh --since "today"`.

Если задачи решаются за 2–5 минут с подсказками по синтаксису — это нормально: важнее, что ты умеешь декомпозировать и проверить результат.
