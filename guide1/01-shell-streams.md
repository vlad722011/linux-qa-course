# Guide 01: Shell и потоки (stdin/stdout/stderr) — реальный разбор (QA)

---

## Цель главы
После этой главы ты должен уметь:
- Отличать терминал от shell и понимать, что shell делает “под капотом”.
- Понимать stdin/stdout/stderr и применять редиректы.
- Собирать простые диагностические отчёты для баг-репортов и CI.

---

## 1) Почему Linux и зачем он QA
Linux — свободная, многопользовательская, многозадачная ОС с открытым исходным кодом.  

Используется:
- на серверах и VPS  
- в CI/CD и контейнерах  
- в backend-инфраструктуре  

Ключевая идея: **всё в Linux — файл или процесс**  

Для QA важно:
- наблюдать поведение приложений на уровне системы  
- анализировать логи, процессы и ресурсы  
- работать без GUI — только терминал  

**Вывод:** Linux — инструмент, shell — язык общения с ним.

---

## 2) Терминал, shell и сессии
**Терминал** — окно, SSH-клиент или вкладка IDE, через которое мы работаем с shell.  

**Shell** (bash на серверах) — программа, которая:
- читает команду  
- создаёт процесс  
- подключает stdin/stdout/stderr  
- управляет окружением  
- возвращает exit code  

**Сессия** — отдельное подключение:
- свой текущий каталог  
- свои переменные окружения  
- свои процессы и история команд  

Каждое SSH-подключение = новая сессия.

---

## 3) Практика: проверить сессию

~~~bash
whoami        # имя текущего пользователя
pwd           # текущий каталог
tty           # терминальное устройство
echo "$SHELL" # какой shell используется
ps            # список процессов
who           # кто подключён
w             # кто подключён + что делает
~~~

---

## 4) Процессы и команды
Когда вводишь команду, например `ls -l`:

Shell:
- создаёт процесс `ls`  
- подключает stdin/stdout/stderr  
- ждёт завершения  
- получает exit code  

**Вывод:** команда = обычный процесс с потоками данных, никакой “магии”.

---

## 5) Потоки данных
У процесса есть 3 стандартных потока:  

| Поток  | Назначение                          |
|--------|------------------------------------|
| stdin  | вход (клавиатура или другой процесс)|
| stdout | обычный вывод                       |
| stderr | ошибки и диагностика                |

**QA-почему важно:**  
- В CI stdout и stderr могут идти отдельно  
- Можно сохранять ошибки отдельно для анализа  

---

## 6) Редиректы (перенаправление потоков)

**stdout → файл:**
~~~bash
echo "hello" > out.txt
echo "world" >> out.txt
cat out.txt
~~~

**stderr → файл:**
~~~bash
ls /no_such_dir 2> err.txt
cat err.txt
~~~

**stdout + stderr → один файл:**
~~~bash
ls /tmp /no_such_dir > all.txt 2>&1
# порядок важен: сначала stdout, потом 2>&1
~~~

**Сокращение bash:**  
~~~bash
ls /tmp /no_such_dir &> all.txt
~~~
*(может не работать в /bin/sh)*

---

## 7) Пайпы (|)

Соединяем stdout одной команды со stdin другой:

~~~bash
cat /var/log/syslog | grep ssh | tail -n 20
~~~

Что происходит:
- `cat` читает файл → stdout  
- `grep` фильтрует строки  
- `tail` выводит последние 20 строк  

---

## 8) Базовые команды

| Команда | Что делает | Полезные флаги |
|---------|------------|----------------|
| pwd     | текущий каталог | — |
| cd      | переход в каталог | .., - |
| ls      | показать содержимое | -l, -a, -h, -R |
| echo    | вывести текст | -n |
| man     | справка | / для поиска |

**Практика:**
~~~bash
cd /home
ls -lah
pwd
echo "Hello Linux"
~~~

---

## 9) QA-кейс: собрать диагностику

~~~bash
mkdir -p ~/qa-diag
cd ~/qa-diag

( echo "== DATE =="; date;
  echo "== UPTIME =="; uptime;
  echo "== DISK =="; df -h ) > out.log 2> err.log

ls -lah
cat out.log
cat err.log
~~~

---

## 10) Примеры для практики

1. Разделить stdout и stderr:  
~~~bash
ls /tmp /no_such_dir > out.txt 2> err.txt
~~~

2. Объединить stdout+stderr:  
~~~bash
ls /tmp /no_such_dir > all.txt 2>&1
~~~

3. Сохранить ошибки SSH:  
~~~bash
sudo grep "Failed" /var/log/auth.log | tail -n 50 > failed_ssh.log
~~~

---

## 11) Мини-инцидент

Симптом: CI упал, в логе только `Command failed`.  

**Действия:**  
- запустить команду с редиректом stderr → файл  
- приложить `err.log` к тикету  
- гипотезы: нет прав, отсутствует файл, сеть  

---

## 12) Итог

- Linux — инструмент, shell — язык  
- Команды = процессы + потоки  
- stdin/stdout/stderr — основа диагностики  
- Редиректы и пайпы — ключ к анализу логов  
- Для QA — базовый навык работы с CI и серверами
